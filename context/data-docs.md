# [Full Market Depth](https://dhanhq.co/docs/v2/full-market-depth/)

Level 3 data includes market depth up to 20 levels. Dhan is extending beyond that and adding 200 level data. This gives a complete picture of market movements and is streamed in real time via WebSockets.

This data can be used to detect demand/supply zones outside of 5-level market depth and build trading systems to detect market movements.

> Only **NSE Equity** and **Derivatives** segments are enabled for Full Market Depth. [Source](https://dhanhq.co/docs/v2/full-market-depth/)

Similar to [Live Market Feed](https://dhanhq.co/docs/v2/live-market-feed/), all request messages over WebSocket are in JSON whereas all response messages over WebSocket are in Binary.

## Establishing Connection

### 20 Level

To establish connection with DhanHQ WebSocket for 20 Level Market Depth, connect to:

```text
wss://depth-api-feed.dhan.co/twentydepth?token=eyxxxxx&clientId=100xxxxxxx&authType=2
```

### 200 Level

To establish connection with DhanHQ WebSocket for 200 Level Market Depth, connect to:

```text
wss://full-depth-api.dhan.co/twohundreddepth?token=eyxxxxx&clientId=100xxxxxxx&authType=2
```

### Query Parameters

| Field | Description |
| --- | --- |
| `token` _required_ | Access Token generated via Dhan |
| `clientId` _required_ | User specific identification generated by Dhan |
| `authType` _required_ | `2` by default |

## Adding Instruments

### 20 Level

For 20 Level Market Depth, you can subscribe up to **50 instruments** in a single connection and receive market data packets.

You can send all 50 instruments in a single JSON message for 20 Depth. You can also send multiple messages over a single connection to subscribe in parts.

#### Request Structure

```json
{
    "RequestCode" : 23,
    "InstrumentCount" : 1,
    "InstrumentList" : [
        {
            "ExchangeSegment" : "NSE_EQ",
            "SecurityId" : "1333"
        }
    ]
}
```

#### Parameters

| Field | Type | Description |
| --- | --- | --- |
| `RequestCode` _required_ | int | Code for subscribing to particular data mode. `23` for Full Market Depth. Refer to [feed request code](https://dhanhq.co/docs/v2/annexure/#feed-request-code) |
| `InstrumentCount` _required_ | int | Number of instruments to subscribe from this request |
| `InstrumentList.ExchangeSegment` _required_ | enum string | Exchange Segment of instrument to be subscribed as found in [Annexure](https://dhanhq.co/docs/v2/annexure/#exchange-segment) |
| `InstrumentList.SecurityId` _required_ | string | Exchange standard ID for each scrip. Refer [here](https://dhanhq.co/docs/v2/instruments/) |

### 200 Level

In 200 level market depth, only **1 instrument per connection** can be subscribed. The JSON payload is sent after the socket connection is established.

#### Request Structure

```json
{
    "RequestCode" : 23,
    "ExchangeSegment" : "NSE_EQ",
    "SecurityId" : "1333"
}
```

#### Parameters

| Field | Type | Description |
| --- | --- | --- |
| `RequestCode` _required_ | int | Code for subscribing to particular data mode. `23` for Full Market Depth. Refer to [feed request code](https://dhanhq.co/docs/v2/annexure/#feed-request-code) |
| `ExchangeSegment` _required_ | enum string | Exchange Segment of instrument to be subscribed as found in [Annexure](https://dhanhq.co/docs/v2/annexure/#exchange-segment) |
| `SecurityId` _required_ | string | Exchange standard ID for each scrip. Refer [here](https://dhanhq.co/docs/v2/instruments/) |

## Keeping Connection Alive

To keep the WebSocket connection alive and prevent it from closing, the server uses a **Ping-Pong** module. The server sends ping every 10 seconds to the client system.

An automated pong is sent by most WebSocket libraries and can be used as the response.

> If the client server does not respond for more than 40 seconds, the connection is closed from the server side and must be re-established.

## Response Structure

The market depth data is sent as a structured binary packet. It requires parsing to a readable format.

All responses consist of a **Response Header** and **Payload**. The header remains the same across responses, while the payload differs.

### 20 Level

The response header is **12 bytes**:

| Bytes | Type | Size | Description |
| --- | --- | --- | --- |
| `1-2` | int16 | `2` | Message Length of the entire payload packet |
| `3` | `[] byte` | `1` | Feed Response Code. Refer [Annexure](https://dhanhq.co/docs/v2/annexure/#feed-response-code) |
| `4` | `[] byte` | `1` | Exchange Segment. Refer [Annexure](https://dhanhq.co/docs/v2/annexure/#exchange-segment) |
| `5-8` | int32 | `4` | Security ID. Refer [here](https://dhanhq.co/docs/v2/instruments/) |
| `9-12` | uint32 | `4` | Message Sequence (to be ignored) |

#### Depth Packet

20-level market depth sends **bid** and **ask** packets separately, each containing **20 packets of 16 bytes each**.

| Bytes | Type | Size | Description |
| --- | --- | --- | --- |
| `0-12` | `[] array` | `12` | Response Header. `41` for Bid Data (Buy), `51` for Ask Data (Sell). Refer [enum](https://dhanhq.co/docs/v2/annexure/#feed-response-code) |
| `13-332` | Bid/Ask Depth Structure | `320` | 20 packets of 16 bytes each |

Each of the 20 packets:

| Bytes | Type | Size | Description |
| --- | --- | --- | --- |
| `1-8` | float64 | `8` | Price |
| `9-12` | uint32 | `4` | Quantity |
| `13-16` | uint32 | `4` | Number of Orders |

### 200 Level

The response header is also **12 bytes**:

| Bytes | Type | Size | Description |
| --- | --- | --- | --- |
| `1-2` | int16 | `2` | Message Length of the entire payload packet |
| `3` | `[] byte` | `1` | Feed Response Code. Refer [Annexure](https://dhanhq.co/docs/v2/annexure/#feed-response-code) |
| `4` | `[] byte` | `1` | Exchange Segment. Refer [Annexure](https://dhanhq.co/docs/v2/annexure/#exchange-segment) |
| `5-8` | int32 | `4` | Security ID. Refer [here](https://dhanhq.co/docs/v2/instruments/) |
| `9-12` | uint32 | `4` | Number of rows to be read for response |

#### Depth Packet

200-level market depth is structured similarly, with bid and ask data sent separately.

| Bytes | Type | Size | Description |
| --- | --- | --- | --- |
| `0-12` | `[] array` | `12` | Response Header. `41` for Bid Data (Buy), `51` for Ask Data (Sell). Refer [enum](https://dhanhq.co/docs/v2/annexure/#feed-response-code) |
| `13-3212` | Bid/Ask Depth Structure | `3200` | 200 packets of 16 bytes each |

Each of the 200 packets:

| Bytes | Type | Size | Description |
| --- | --- | --- | --- |
| `1-8` | float64 | `8` | Price |
| `9-12` | uint32 | `4` | Quantity |
| `13-16` | uint32 | `4` | Number of Orders |

> Whenever 20 or 200 level depth packets are sent on the connection, they are stacked one after another in a single message. For 20 level depth, if two instruments are subscribed, the first instrument's Bid packet followed by Ask packet is added first, then the second instrument’s Bid and Ask packets in the same sequence. You can break down the packet on the basis of length.

## Feed Disconnect

To disconnect WebSocket:

```json
{
    "RequestCode" : 12
}
```

If the WebSocket is disconnected from the server side, you will receive a disconnection packet containing a reason code.

- If more than **5 websockets** are established, the first socket will be disconnected with code `805` for every additional connection.

| Bytes | Type | Size | Description |
| --- | --- | --- | --- |
| `0-12` | `[] array` | `8` | Response Header with code `50`. Refer [enum](https://dhanhq.co/docs/v2/annexure/#feed-response-code) |
| `13-14` | int16 | `2` | Disconnection message code. Refer [here](https://dhanhq.co/docs/v2/annexure/#data-api-error) |

Detailed disconnection message codes are available [here](https://dhanhq.co/docs/v2/annexure/#data-api-error). [Source](https://dhanhq.co/docs/v2/full-market-depth/)

# [Market Quote](https://dhanhq.co/docs/v2/market-quote/)

This API gives snapshots of multiple instruments at once. You can fetch **LTP**, **Quote**, or **Market Depth** of instruments via API, returning real-time data at the time of request. [Source](https://dhanhq.co/docs/v2/market-quote/)

| Method | Endpoint | Description |
| --- | --- | --- |
| `POST` | `/marketfeed/ltp` | Get ticker data of instruments |
| `POST` | `/marketfeed/ohlc` | Get OHLC data of instruments |
| `POST` | `/marketfeed/quote` | Get market depth data of instruments |

> You can fetch up to **1000 instruments** in a single API request with a rate limit of **1 request per second**.

## Ticker Data

Retrieve LTP for a list of instruments with a single API request.

```bash
curl --request POST \
--url https://api.dhan.co/v2/marketfeed/ltp \
--header 'Accept: application/json' \
--header 'Content-Type: application/json' \
--header 'access-token: JWT' \
--header 'client-id: 1000000001' \
--data '{}'
```

### Headers

| Header | Description |
| --- | --- |
| `access-token` _required_ | Access Token generated via Dhan |
| `client-id` _required_ | User specific identification generated by Dhan |

### Request Structure

```json
{
    "NSE_EQ":[11536],
    "NSE_FNO":[49081,49082]
}
```

### Parameters

| Field | Field Type | Description |
| --- | --- | --- |
| [Exchange Segment ENUM](https://dhanhq.co/docs/v2/annexure/#exchange-segment) _required_ | array | Security ID - can be found [here](https://dhanhq.co/docs/v2/instruments/) |

### Response Structure

```json
{
    "data": {
        "NSE_EQ": {
            "11536": {

        },

                "last_price": 368.15

```

### Response Parameters

| Field | Type | Description |
| --- | --- | --- |
| `last_price` | float | LTP of the instrument |

# [Live Market Feed](https://dhanhq.co/docs/v2/live-market-feed/)

Real-time market data across exchanges and segments can be received via WebSocket. WebSocket keeps a persistent connection open and allows the server to push tick-by-tick market data to your systems.

All Dhan platforms use these same market feed WebSocket connections.

> You can establish up to **five WebSocket connections per user** with **5000 instruments on each connection**. [Source](https://dhanhq.co/docs/v2/live-market-feed/)

All request messages over WebSocket are in JSON whereas all response messages over WebSocket are in Binary.

Dhan also provides a [Python Library](https://github.com/dhan-oss/DhanHQ-py) to quick-start with Live Market Feed.

## Establishing Connection

```text
wss://api-feed.dhan.co?version=2&token=eyxxxxx&clientId=100xxxxxxx&authType=2
```

### Query Parameters

| Field | Description |
| --- | --- |
| `version` _required_ | `2` for DhanHQ v2 |
| `token` _required_ | Access Token generated via Dhan |
| `clientId` _required_ | User specific identification generated by Dhan |
| `authType` _required_ | `2` by default |

## Adding Instruments

You can subscribe up to **5000 instruments** in a single connection.

> You can only send up to **100 instruments in a single JSON message**. You can send multiple messages over one connection to subscribe to all instruments.

### Request Structure

```json
{
    "RequestCode" : 15,
    "InstrumentCount" : 2,
    "InstrumentList" : [
        {
            "ExchangeSegment" : "NSE_EQ",
            "SecurityId" : "1333"
        },
        {
            "ExchangeSegment" : "BSE_EQ",
            "SecurityId" : "532540"
        }
    ]
}
```

### Parameters

| Field | Type | Description |
| --- | --- | --- |
| `RequestCode` _required_ | int | Code for subscribing to particular data mode. Refer [feed request code](https://dhanhq.co/docs/v2/annexure/#feed-request-code) |
| `InstrumentCount` _required_ | int | Number of instruments to subscribe from this request |
| `InstrumentList.ExchangeSegment` _required_ | enum string | Exchange Segment of instrument to be subscribed as found in [Annexure](https://dhanhq.co/docs/v2/annexure/#exchange-segment) |
| `InstrumentList.SecurityId` _required_ | string | Exchange standard ID for each scrip. Refer [here](https://dhanhq.co/docs/v2/instruments/) |

## Keeping Connection Alive

The server uses **Ping-Pong** to keep the connection alive.

- Server sends ping every 10 seconds.
- WebSocket libraries usually send automated pong.
- If the client does not respond for more than 40 seconds, the server closes the connection.

## Market Data

Live Market Feed is real-time binary data. There are three modes:

- Ticker Data
- Quote Data
- Full Data

### Binary Response

Binary messages are compact and fast, but require parsing.

All responses consist of:

- **Response Header**
- **Payload**

The response header is **8 bytes**:

| Bytes | Type | Size | Description |
| --- | --- | --- | --- |
| `1` | `[] byte` | `1` | Feed Response Code. Refer [Annexure](https://dhanhq.co/docs/v2/annexure/#feed-response-code) |
| `2-3` | int16 | `2` | Message Length of the entire payload packet |
| `4` | `[] byte` | `1` | Exchange Segment. Refer [Annexure](https://dhanhq.co/docs/v2/annexure/#exchange-segment) |
| `5-8` | int32 | `4` | Security ID. Refer [here](https://dhanhq.co/docs/v2/instruments/) |

### Endianness

The data on DhanHQ WebSockets is sent in **Little Endian**.

- **Little Endian**: Least significant byte first
- **Big Endian**: Most significant byte first

If your system is Big Endian, you must define endianness while reading the WebSocket.

## Ticker Packet

Contains **Last Traded Price (LTP)** and **Last Traded Time (LTT)**.

| Bytes | Type | Size | Description |
| --- | --- | --- | --- |
| `0-8` | `[] array` | `8` | Response Header with code `2`. Refer [enum](https://dhanhq.co/docs/v2/annexure/#feed-response-code) |
| `9-12` | float32 | `4` | Last Traded Price |
| `13-16` | int32 | `4` | Last Trade Time (EPOCH) |

| `9-12` | float32 | `4` | Previous day closing price |
| `13-16` | int32 | `4` | Open Interest - previous day |

# [Historical Data](https://dhanhq.co/docs/v2/historical-data/)

This API gives historical candle data for the desired scrip across segments and exchange. Data is presented as candles containing timestamp, open, high, low, close, and volume. [Source](https://dhanhq.co/docs/v2/historical-data/)

| Method | Endpoint | Description |
| --- | --- | --- |
| `POST` | `/charts/historical` | Get OHLC for daily timeframe |
| `POST` | `/charts/intraday` | Get OHLC for minute timeframe |

## Daily Historical Data

Retrieve OHLC and Volume of daily candles for the desired instrument. Data is available back to the date of the instrument’s inception.

```bash
curl --request POST \
--url https://api.dhan.co/v2/charts/historical \
--header 'Content-Type: application/json' \
--header 'access-token: JWT' \
--data '{}'
```

}

## Intraday Historical Data

Retrieve Open, High, Low, Close, OI & Volume of **1, 5, 15, 25 and 60 min** candles for the desired instrument for the last 5 years. This data is available for all exchanges and segments for all active instruments. [Source](https://dhanhq.co/docs/v2/historical-data/)

```bash
curl --request POST \
--url https://api.dhan.co/v2/charts/intraday \
--header 'Accept: application/json' \
--header 'Content-Type: application/json' \
--header 'access-token: ' \
--data '{}'
```

### Request Structure

```json
{
    "securityId": "1333",
    "exchangeSegment": "NSE_EQ",
    "instrument": "EQUITY",
    "interval": "1",
    "oi": false,
    "fromDate": "2024-09-11 09:30:00",
    "toDate": "2024-09-15 13:00:00"
}
```

### Parameters

| Field | Field Type | Description |
| --- | --- | --- |
| `securityId` _required_ | string | Exchange standard ID for each scrip. Refer [here](https://dhanhq.co/docs/v2/instruments/) |

| `instrument` _required_ | enum string | Instrument type of the scrip. Refer [here](https://dhanhq.co/docs/v2/annexure/#instrument) |

| `oi` _optional_ | boolean | Open Interest data for Futures & Options |
| `fromDate` _required_ | string | Start date of the desired range |
| `toDate` _required_ | string | End date of the desired range |
